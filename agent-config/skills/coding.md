# Coding Principles

Follow existing conventions in the repo — naming, structure, patterns. Read before writing.

- Write clear, readable code. Explicit over clever.
- Only do what was asked. No unrequested refactors, no extra features.
- Consider edge cases before marking a task done.
- If something is unclear, ask — don't assume and proceed.
- Don't add comments that just restate the code. Only comment non-obvious logic.
